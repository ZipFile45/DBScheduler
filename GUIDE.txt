============================================================
                 DBScheduler – Руководство пользователя
                          Версия 2.2 (2025)
============================================================

Программа для автоматического резервного копирования файлов 
и баз данных с поддержкой локальных, сетевых и облачных 
хранилищ.

Поддерживаемые ОС: Windows 10/11, Windows Server 2012 R2+

------------------------------------------------------------
1. ВОЗМОЖНОСТИ
------------------------------------------------------------
- Резервное копирование каталогов и файловых баз
- Поддерживаемые форматы архивов:
    ZIP, TAR, TAR.GZ, TAR.BZ2, TAR.XZ, 7Z
- Автоматическое копирование на дополнительные диски/папки
- Автоматическая загрузка архивов в облако (новое в 2.2)
- Гибкое расписание: ежедневно, каждый час, 6/12 часов
- Несколько триггеров расписания
- Email-уведомления об успехах и ошибках
- Автозапуск через Планировщик Windows
- Завершение процессов 1С перед архивацией
- Очистка старых архивов
- Условия запуска: питание, простой, пробуждение ПК
- Повтор задач при ошибке, таймаут, завершение зависших задач
- Локальное хранение конфигураций и логов

------------------------------------------------------------
5. НАСТРОЙКА КОНФИГУРАЦИИ
------------------------------------------------------------

5.1 Основные настройки
----------------------
Источник: путь к базе или каталогу.
Папка архивов: место хранения архивов.
Папка копий: вторичное хранилище.
Закрытие 1С: завершение процессов 1cv8.exe/1cv8c.exe.
Сохраните конфигурацию.

5.2 Вкладка "Облако" (новое в 2.2)
----------------------------------
Поддерживаемые сервисы:
- WebDAV
- S3 (AWS, B2, Wasabi, DigitalOcean и др.)
- Google Drive
- OneDrive / SharePoint
- Dropbox
- Box
- Mega
- pCloud
- Mail.ru Облако

Параметры:
Base URL, логин/пароль, SSL, папка, проверка подключения.

ДЛЯ ЗАГРУЗКИ АРХИВА ТРЕБУЕТСЯ ВРЕМЯ!

5.3 Архив (новые форматы)
-------------------------
Выберите формат: ZIP / TAR / TAR.GZ / TAR.BZ2 / TAR.XZ / 7Z

5.4 Расписание
--------------
Типы расписаний: ежедневно, каждый час, 6/12 часов.
Поддерживаются:
- выполнение при пропуске
- запуск по требованию
- автозапуск через Планировщик

5.5 Email уведомления
----------------------
Параметры SMTP, логин/пароль, получатели.
Тест отправки письма.

5.6 Очистка архивов
-------------------
Оставляет только указанное число последних архивов.

5.7 Условия
-----------
- Питание: запуск только при питании от сети, пробуждение ПК
- Простой: запуск в простое, задержка, остановка при выходе

5.8 Параметры
-------------
- Выполнение по требованию
- Выполнение при пропуске
- Повтор при ошибке (попытки + интервал)
- Ограничение времени выполнения
- Завершение зависших задач

------------------------------------------------------------
9. УСТРАНЕНИЕ НЕПОЛАДОК
------------------------------------------------------------
Проблемы облака:
- Неверные логин/пароль
- Не тот Base URL (WebDAV/S3)
- Ошибка SSL

Проблемы архивов:
- Недостаточно места
- Нет прав доступа

Email:
- Неверный SMTP или порт
- Нужен SSL или STARTTLS

Планировщик:
- Запустите DBScheduler как администратор

------------------------------------------------------------
10. ЧАСТЫЕ ВОПРОСЫ (FAQ)
------------------------------------------------------------
Q: Можно несколько расписаний?
A: Да.

Q: Что делает очистка архивов?
A: Удаляет старые архивы, оставляет последние N.

Q: Что если облако недоступно?
A: Архив создаётся локально, ошибка записывается в лог.

Q: Какой формат выбрать?
A: 7Z — максимальная компрессия, ZIP — совместимость.
